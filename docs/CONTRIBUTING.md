# Contributing to PLV Residences

## Welcome! 👋

Thank you for your interest in contributing to the PLV Residences website. This document provides guidelines and instructions for contributing.

## Getting Started

### 1. Fork the Repository
```bash
git clone https://github.com/plvresidences/plv-residences.git
cd plv-residences
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Edit files as needed
- Follow the coding standards below
- Test your changes locally

### 4. Commit Your Changes
```bash
git add .
git commit -m "[type] Brief description of changes"
```

**Commit Types:**
- `[feat]` - New feature
- `[fix]` - Bug fix
- `[docs]` - Documentation
- `[style]` - Formatting/styling
- `[refactor]` - Code refactoring
- `[test]` - Tests

### 5. Push and Create Pull Request
```bash
git push origin feature/your-feature-name
```

Then create a pull request on GitHub.

## Coding Standards

### HTML
- Use semantic HTML5 tags
- Maintain proper indentation (2 spaces)
- Include alt text for all images
- Use descriptive class names

### CSS
- Use CSS3 features
- Mobile-first approach
- Follow BEM methodology for class naming
- Keep specificity low
- Use CSS variables for colors and fonts

### JavaScript
- Use ES6+ syntax
- Write clean, readable code
- Add comments for complex logic
- Avoid global variables
- Use const by default, let when needed

### Accessibility
- Ensure WCAG 2.1 Level AA compliance
- Test with screen readers
- Maintain color contrast ratios
- Provide keyboard navigation

## Pull Request Process

1. **Update documentation** if needed
2. **Link to related issues** in the PR description
3. **Request review** from maintainers
4. **Address feedback** promptly
5. **Ensure all checks pass** before merging

## Reporting Issues

### Bug Report
Include:
- Clear description
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots/videos if applicable
- Browser/device info

### Feature Request
Include:
- Clear description of the feature
- Use case/problem it solves
- Proposed solution
- Alternative approaches

## Code Review

All contributions require code review. Reviewers will check for:
- Code quality and style
- Functionality and correctness
- Performance implications
- Security concerns
- Documentation completeness
